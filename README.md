# نظام حجز مواعيد العيادات الطبية

نظام كامل لحجز المواعيد الطبية: **Backend** بلغة Python (FastAPI) + **Frontend**
منفصل تمامًا (HTML/CSS/JS بدون إطار عمل) + قاعدة بيانات قابلة للتوسع
(SQLite للتطوير، وقابلة للتحويل إلى PostgreSQL/MySQL بسطر واحد).

يدعم اللغتين **العربية والتركية** (RTL/LTR تلقائيًا)، بتصميم عصري بسيط
(Light Mode) مع رسوم متحركة (animations) خفيفة على الأجهزة.

## هيكل المشروع

```
clinic-booking-system/
├── backend/     # FastAPI REST API — راجع backend/README.md للتفاصيل
└── frontend/    # موقع الحجز + لوحة التحكم — راجع frontend/README.md للتفاصيل
```

الـ Backend والـ Frontend منفصلان تمامًا ويتواصلان فقط عبر REST API
(‏`/api/v1/...`)، بحيث يمكنك مستقبلًا بناء تطبيق جوال (Flutter / React
Native / Swift / Kotlin) يستخدم نفس الـ API دون أي تعديل على الخادم.

## تشغيل المشروع خلال دقائق

**1) شغّل الـ Backend:**
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m app.seed          # ينشئ حساب مدير افتراضي + بيانات تجريبية
uvicorn app.main:app --reload
```
سيعمل على: `http://127.0.0.1:8000` — وتوثيق الـ API تلقائيًا على `/docs`

**2) شغّل الـ Frontend (في نافذة طرفية أخرى):**
```bash
cd frontend
python3 -m http.server 5500
```
افتح المتصفح على: `http://127.0.0.1:5500`

**بيانات دخول المدير الافتراضية** (غيّرها من ملف `backend/.env` قبل الإطلاق):
```
admin@myclinic.com / Admin@12345
```

## الميزات المُنفّذة

- **تسجيل دخول وتسجيل حساب** للمرضى (JWT)، مع صلاحيات منفصلة للمدير.
- **إدارة الأقسام الطبية** (اسم/وصف/أيقونة بالعربي والتركي) من لوحة التحكم.
- **إدارة الأطباء** لكل قسم (الاسم، المسمى الوظيفي، نبذة، صورة).
- **جدول عمل أسبوعي لكل طبيب** (أيام + ساعات + مدة الموعد) + أيام إجازة.
- **حجز المواعيد**: القسم ← الطبيب ← التاريخ/الوقت المتاح فعليًا ← تأكيد.
  الأوقات المتاحة تُحسب لحظيًا من جدول الطبيب مطروحًا منها الحجوزات
  والإجازات، فلا يمكن حجز نفس الموعد مرتين.
- **صفحة "مواعيدي"** للمريض: عرض المواعيد القادمة/السابقة وإلغاء الحجز.
- **لوحة تحكم المدير**: نظرة عامة، كل الحجوزات (مع فلاتر وتغيير الحالة)،
  إدارة الأطباء والأقسام والجداول.
- **دعم لغتين** (عربي RTL / تركي LTR) قابل للتوسعة بسهولة للغة ثالثة.
- **تصميم متجاوب** خفيف على الأجهزة مع حركات (fade/slide/hover) هادئة.

## التخصيص لعيادتك (بدون كتابة كود تقريبًا)

| ما تريد تغييره | من أين |
|---|---|
| اسم العيادة | `frontend/assets/js/config.js` |
| الشعار/الصور | `frontend/assets/js/config.js` + `frontend/assets/images/` |
| الألوان والخطوط | متغيرات CSS أعلى `frontend/assets/css/style.css` |
| الأقسام/الأطباء/الجداول | مباشرة من لوحة تحكم `/admin.html` بعد تسجيل الدخول كمدير |
| نصوص الواجهة | `frontend/assets/i18n/ar.json` و `tr.json` |
| رابط الخادم | `frontend/assets/js/config.js` → `API_BASE_URL` |

## التوسع مستقبلاً

- **قاعدة البيانات**: غيّر `DATABASE_URL` في `backend/.env` من SQLite إلى
  PostgreSQL أو MySQL دون تعديل أي كود (راجع `backend/README.md`).
- **تطبيق الجوال**: بما أن كل شيء عبر REST API موثّق تلقائيًا على
  `/docs`، يمكن لأي تطبيق جوال استهلاك نفس نقاط النهاية (endpoints)
  المستخدمة في هذا الموقع.
- **صفحات/أقسام جديدة في اللوحة**: بنية الملفات مقسّمة بشكل يسهّل الإضافة
  (راوتر جديد في الـ backend + صفحة/ملف JS جديد في الـ frontend).

## ملاحظات إنتاجية قبل الإطلاق الفعلي

- غيّر `SECRET_KEY` وبيانات المدير الافتراضية في `backend/.env`.
- فعّل HTTPS، وحدّد `CORS_ORIGINS` على نطاق موقعك الفعلي فقط.
- استخدم قاعدة بيانات حقيقية (PostgreSQL) بدلًا من SQLite، مع نظام
  migrations (مثل Alembic) بدلًا من `create_all()`.
